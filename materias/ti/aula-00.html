Hierarquia DIKW (Dados, Informação, Conhecimento, Inteligência)Diferença entre Dado e InformaçãoQualidade de DadosCiclo de Vida da InformaçãoSemântica e Contexto

A Hierarquia DIKW estabelece a progressão de dados (elementos brutos sem significado) para informação (dados contextualizados), conhecimento (compreensão estruturada) e inteligência (sabedoria aplicada). Dados são fatos objetivos desvinculados de contexto, enquanto informação é dado processado com significado. A qualidade de dados envolve completude, consistência, precisão e oportunidade. Contexto e semântica são fundamentais para transformar dados em informação útil para tomada de decisão organizacional.


Pegadinhas da Banca

Confundir dado com informação - dado é bruto, informação é contextualizada
Esquecer que dado sem contexto não transmite significado
Assumir que qualidade envolve apenas completude (precisa considerar consistência, precisão e oportunidade)
Não diferenciar entre conhecimento e informação - conhecimento requer compreensão estruturada

Coleta e Captura de DadosArmazenamento e PersistênciaProcessamento e Transformação (ETL)Análise e RecuperaçãoArquivamento e Exclusão

O ciclo de vida de dados abrange coleta, armazenamento, processamento, análise e arquivamento. O processo ETL (Extração, Transformação, Carga) é fundamental para preparar dados em data warehouses. O staging area (área intermediária) armazena dados coletados antes do processamento. Cada fase requer governança adequada, desde a origem até o descarte ou arquivamento permanente dos dados.


Pegadinhas da Banca

Esquecer a fase de ETL no processo de data warehouse
Confundir staging area com data warehouse final
Não considerar o arquivamento como fase essencial do ciclo
Ignorar a necessidade de governança em cada etapa do ciclo

Definição e Função de MetadadosMetadados em Documentos (Office, BrOffice)Metadados em Bancos de DadosImportância para RastreabilidadeTipos de Metadados (descritivos, estruturais, administrativos)

Metadados são dados sobre dados - informações que descrevem características de outros dados. Arquivos de processadores de texto (Microsoft Office, BrOffice) contêm metadados como título, autor, palavras-chave e histórico de revisões. Metadados são essenciais para rastreabilidade, catalogação e recuperação de dados. Existem três tipos principais: descritivos (identificação), estruturais (organização) e administrativos (gestão).


Pegadinhas da Banca

Pensar que metadados só existem em bancos de dados - documentos também os possuem
Confundir metadados com dados reais
Negligenciar metadados em auditorias de segurança (contêm histórico sensível)
Não reconhecer que metadados facilitam governança de dados

4Dados Estruturados e Não-EstruturadosNão especificado
14 questõesCEBRASPE 9FGV 5
Dados Estruturados (schemas rígidos)Dados Semiestruturados (XML, JSON)Dados Não-Estruturados (textos, imagens, vídeos)SQL para Dados EstruturadosDesafios de Processamento

Dados estruturados seguem modelos rígidos (bancos relacionais, consultáveis por SQL). Dados semiestruturados possuem marcações ou identificadores (XML, JSON), permitindo processamento parcial. Dados não-estruturados (textos, imagens, áudio, vídeo) carecem de formato predefinido e exigem técnicas avançadas de análise. A maioria dos dados corporativos modernos é semiestruturada ou não-estruturada, representando desafio maior de processamento.


Pegadinhas da Banca

Esquecer que dados semiestruturados existem como categoria intermediária
Achar que todos os dados não-estruturados são impossíveis de processar
Confundir JSON/XML com dados não-estruturados - são semiestruturados
Ignorar que dados estruturados nem sempre são mais valiosos que não-estruturados

13 questõesCEBRASPE 11FGV 2
Definição e Princípios de Dados AbertosLicenças e ReutilizaçãoGoverno Aberto e Transparência5 Estrelas de Dados AbertosBenefícios e Limitações

Dados abertos são informações livremente acessíveis, reutilizáveis e redistribuíveis sob licenças apropriadas. Diferem de dados móveis (dados em dispositivos móveis) ou dados públicos (apenas acessíveis). O modelo 5 Estrelas de Dados Abertos estabelece níveis de qualidade desde formatos proprietários até dados vinculados (linked data). Promovem transparência governamental, inovação e responsabilidade pública, mas exigem cuidado com privacidade.


Pegadinhas da Banca

Confundir dados abertos com dados móveis ou dados públicos
Pensar que dados abertos significam sem qualquer restrição de privacidade
Ignorar a importância de licenças apropriadas para dados abertos
Não reconhecer que dados abertos exigem metadados claros para serem úteis
